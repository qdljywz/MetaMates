#### 终端功能技术方案

**核心技术栈**：
- **xterm.js** - 终端 UI 渲染库，提供真正的终端体验
- **node-pty** - 伪终端进程管理，支持交互式程序
- **FitAddon** - 终端自适应大小

**终端架构**：
```
┌─────────────────────────────────────────────────────────┐
│                    TerminalPanel.tsx                     │
│  ┌─────────────────────────────────────────────────────┐ │
│  │                  xterm.js Terminal                   │ │
│  │  - 渲染终端 UI                                        │ │
│  │  - 处理用户输入                                       │ │
│  │  - 显示输出                                          │ │
│  └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
                           │
                           │ IPC
                           ▼
┌─────────────────────────────────────────────────────────┐
│                     main.ts (Electron)                   │
│  ┌─────────────────────────────────────────────────────┐ │
│  │                   node-pty                           │ │
│  │  - 启动 shell 进程 (PowerShell/Bash)                 │ │
│  │  - 管理 PTY 会话                                     │ │
│  │  - 处理输入/输出流                                    │ │
│  └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**IPC 接口**：
```typescript
terminal: {
  start: (cwd?: string) => Promise<{ success: boolean; pid?: number }>
  input: (pid: number, data: string) => Promise<{ success: boolean }>
  resize: (pid: number, cols: number, rows: number) => Promise<{ success: boolean }>
  kill: (pid: number) => Promise<{ success: boolean }>
  onOutput: (callback: (data: TerminalOutput) => void) => void
}
```

**相关文件**：
- `src/components/TerminalPanel.tsx` - xterm.js 终端面板组件
- `src/components/ChatPanel.tsx` - 添加模式切换逻辑
- `electron/main.ts` - node-pty 进程管理
- `electron/preload.cjs` - 暴露终端 API
- `src/types/electron.d.ts` - TypeScript 类型定义

## 二、设计原则

1. **右键菜单差异化**：文件夹和文件显示不同的操作选项
2. **功能完整性**：已有功能应完整可用，不应显示"开发中"
3. **用户反馈**：操作后应有明确的视觉反馈（message 提示）
4. **测试验证**：修改后必须进行 TypeScript 编译、单元测试、应用启动测试

**CLI崩溃问题（非代码问题）**

测试发现所有CLI在启动时崩溃，原因是CLI本身的内存问题：

| CLI | 错误信息 | 原因 |
|-----|---------|------|
| Gemini CLI | "JavaScript heap out of memory" | CLI进程内存不足 |
| Claude Code | "JavaScript heap out of memory" | CLI进程内存不足 |
| CodeBuddy | "The system cannot execute the specified program" | 路径问题 |

**重要发现**：
- POC也遇到同样的崩溃问题，说明这不是代码问题
- CLI崩溃发生在 `initialize` 请求之后，`session/new` 请求期间
- 这是CLI本身的资源限制问题，不是ACP集成代码的问题

**解决方案**：
1. 增加CLI进程的内存限制（通过Node.js `--max-old-space-size` 参数）
2. 或者等待CLI更新修复内存问题
3. 或者联系CLI开发者报告问题

**代码重写完成**：
- `AgentChatPanel.tsx` - 完全按照POC的index.html重写
- `preload.cjs` - 更新IPC接口匹配POC
- `ipcHandlers.ts` - 保持POC的设计
- `AcpConnection.ts` - 保持POC的设计

**重要设计原则（必须遵守）**

1. **每个CLI有独立的sessionId**：不要使用单一的sessionId state，必须从connectionStatuses Map中获取
2. **不要擅自添加POC没有的功能**：迁移时严格按照POC设计，不要自己创造
3. **状态按backend独立存储**：connectionStatuses、messagesMap等都是Map结构，按backend分开

```typescript
const extractText = useCallback((content: any): string => {
  if (!content) return ''
  if (typeof content === 'string') return content
  if (content.type === 'text' && content.text) return content.text
  if (content.type === 'content' && content.content) {
    return extractText(content.content)
  }
  if (content.content) {
    if (typeof content.content === 'string') return content.content
    if (content.content.type === 'text' && content.content.text) return content.content.text
  }
  if (Array.isArray(content)) {
    return content.map(c => extractText(c)).join('')
  }
  return ''
}, [])
```

```typescript
function syncDir(srcDir: string, destDir: string, relativePath: string = '') {
  const items = fs.readdirSync(srcDir, { withFileTypes: true })
  
  for (const item of items) {
    const srcItemPath = path.join(srcDir, item.name)
    const destItemPath = path.join(destDir, item.name)
    const itemRelativePath = relativePath ? `${relativePath}/${item.name}` : item.name
    
    if (item.isDirectory()) {
      if (!fs.existsSync(destItemPath)) {
        fs.mkdirSync(destItemPath, { recursive: true })
      }
      syncDir(srcItemPath, destItemPath, itemRelativePath)
    } else {
      if (!fs.existsSync(destItemPath)) {
        fs.copyFileSync(srcItemPath, destItemPath)
        createdItems.push(itemRelativePath)
      } else {
        skippedItems.push(itemRelativePath)
      }
    }
  }
}
```
