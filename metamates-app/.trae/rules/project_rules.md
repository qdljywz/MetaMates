## 核心原则

### 1. 不说谎、不糊弄
- **禁止声称"完成"但实际未验证**
- **禁止输出"验证成功"但实际未检查**
- **禁止声称"文件已保存"但实际未保存**
- **禁止用"测试通过"糊弄用户**

### 2. 真实验证
- 每次修改代码后，必须运行应用测试
- 必须检查实际结果，不能只看代码
- 必须验证文件是否真的被创建/更新
- 必须验证功能是否真的工作

### 3. 诚实沟通
- 发现问题要诚实承认
- 不确定时要明确说明
- 没有完成要如实告知
- 不要用"应该可以"来糊弄

## 功能确认机制

### 触发条件
当用户说 **"功能确认"** 时，必须执行以下操作：

### 执行步骤
1. **总结近期对话**：回顾最近 3-5 轮对话中讨论的功能
2. **提取共识**：识别已达成共识的功能定义和变更
3. **更新文档**：更新 `e:\Trae\Metamates\功能确认.md` 文件

### 文档格式
功能确认文档只记录功能对齐内容，包括：
- 核心模块定义（功能清单 + 状态）
- 设计原则
- 功能变更记录（按日期）

### 注意事项
- 只记录已确认的功能，不记录待讨论的内容
- 状态标记：✅ 已实现 / ⚠️ 部分实现 / ❌ 未实现
- 每次更新时追加"功能变更记录"，不要删除历史记录

## 开发流程

### 1. 代码修改后必须测试
- 每次修改代码后，运行 `npm run electron:compile` 与相关单测
- 涉及 Agent/CLI 时确认 `AcpDetector` 日志
- 确保应用正常启动，界面正常显示

### 2. TypeScript 编译检查
- 修改代码后，运行 `npx tsc --noEmit` 检查类型错误
- 修复所有编译错误后再交付

### 3. 功能测试清单
- [ ] 应用正常启动
- [ ] 侧边栏文件列表正常显示
- [ ] 打开工作区功能正常
- [ ] Agent CLI 自动发现（工具栏图标）
- [ ] slash 命令与对话正常
- [ ] 自动保存功能正常

## 代码规范

### 1. 图标使用
- 使用 Ant Design 图标时，确保图标名称正确

### 2. 文件操作
- 创建文件时确保文件名包含 `.md` 扩展名
- 使用 `resolveWorkspaceFilePath` 等统一路径解析

### 3. 错误处理
- 所有异步操作都要有 try-catch 或错误处理
- 用户操作失败时显示友好的错误提示
