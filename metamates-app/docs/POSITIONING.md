# MetaMates 产品定位

> 定稿：2026-06-20 · **双核叙事**：私人灵感仓库 + 引擎  
> 受众：产品、开发、对外介绍。与 [PERSONAL_SCOPE.md](./PERSONAL_SCOPE.md)（边界清单）配套阅读。

---

## 一句话

**MetaMates = 私人灵感仓库 + 思考引擎。**  
灵感随手进仓库；和 AI 对话，引擎从仓库里读出联系、升维想法、写出计划——结果落回本地文件，不消失在聊天里。

---

## 双核定位

MetaMates 不是「笔记软件 + 聊天挂件」，而是两个同等重要的部分：

| | **私人灵感仓库** | **引擎** |
|---|------------------|----------|
| **是什么** | 你电脑上的一个文件夹，专门装碎片、日记、灵感、项目笔记 | 右侧 AI 对话 + 方法论命令（/today、/trace、/graduate…） |
| **解决什么** | 「想法到处散，怕丢、怕乱、怕上传云端」 | 「仓库里的东西太散，需要有人帮读、帮连、帮想、帮写」 |
| **用户动作** | 剪藏、随手记、日记、双链整理（低门槛，不必先想清楚） | **主入口：和 AI 说话**；或点一条 slash 命令 |
| **产物** | Markdown 文件、Inbox、永久笔记、Master Control | 计划、复盘、灵感升级、关联报告——**写回仓库** |
| **隐喻** | 私人库房：先进先出不必，但都在你名下 | 加工车间：读原料、出成品，成品仍回库房 |

```text
        手机剪藏 / 随手记 / 日记
                 ↓
    ┌────────────────────────────┐
    │     私人灵感仓库            │  ← 本地文件夹，Markdown，你拥有
    │  Inbox · 日记 · 永久笔记    │
    └─────────────┬──────────────┘
                  │ 读 / 写
                  ↓
    ┌────────────────────────────┐
    │          引 擎              │  ← 主入口：右侧 AI 对话
    │  对话 · slash · 多 CLI      │     /today /trace /graduate …
    └─────────────┬──────────────┘
                  │ 成品落盘
                  ↓
           计划 · 洞察 · 升级后的笔记
```

**关键关系（避免说反）：**

- 仓库 **不是** 主界面上的「先写一大篇再谈 AI」——那是传统笔记软件的路径。
- 引擎 **才是** 日常打开 App 后的第一动作：**选 AI → 说话或下命令**。
- 中间编辑器 **是** 看成品、改细节的地方，不是产品的唯一主场。

## 我们是谁 / 不是谁

| 我们是 | 我们不是 |
|--------|----------|
| **私人灵感仓库**（本地 Markdown，零强制结构） | 云端笔记 SaaS、协作办公套件 |
| **思考引擎**（对话 + 13 条方法论命令，读写同一仓库） | 纯 Chat 客户端（聊完即散，如独立 AionUi） |
| **本地 Electron 桌面应用** | 用浏览器打开的多租户网站 |
| 外接本机 AI（Gemini / Claude / CodeBuddy / Ollama） | 内置大模型 API 聚合商 |
| 灵感 → 洞察 → 行动 的闭环 | 只存不加工的「笔记坟墓」 |

> 架构说明见 [DESKTOP_APP.md](./DESKTOP_APP.md)。手机 `/mobile` 只负责**往仓库里丢灵感**（剪藏），**不是**引擎的主入口。

---

## 核心用户（默认假设）

1. **首发用户**：有持续灵感/日记/项目笔记，但懒得整理的人（创作者、知识工作者、自我成长者）。
2. **体验目标**：打开 App → 右侧和 AI 说一句话就能开工；不必先学会文件夹结构或 Prompt。
3. **不优先**：企业 IT、SSO、多人协作、插件生态。

## 产品支柱

```text
┌─────────────────────────────────────────┐
│  私人灵感仓库                             │
│  本地 Markdown · 双链 · 标签 · 搜索 · 图谱 │
│  Inbox 剪藏 · 日记 · 永久笔记 · 项目       │
├─────────────────────────────────────────┤
│  引擎                                    │
│  AI 对话（主入口）· slash 命令 · 多 CLI    │
│  读仓库 · 写回文件 · 工具权限 · 持续对话    │
├─────────────────────────────────────────┤
│  桥接层                                  │
│  Vault API · MCP · 移动端剪藏 · Ollama    │
└─────────────────────────────────────────┘
```

- **仓库负责「存」**：降低捕获摩擦，允许乱、允许碎。
- **引擎负责「化」**：把碎片变成计划、关联、升级笔记；**对话是最短的启动方式**。

## 界面原则（2026-06）

MetaMates **不是**独立 Chat App 三栏布局，也**不是**「编辑器占 80% 屏幕的传统笔记」。

```text
┌──────────┬─────────────────────────┬──────────────────┐
│ 仓库导航  │  编辑器（看 & 改成品）    │  引擎（主入口）    │
│ 文件树    │  多标签 Markdown         │  AI 对话 + 语音    │
│ 日历/搜索 │  双链 · 标签 · 自动保存   │  slash · CLI 切换  │
└──────────┴─────────────────────────┴──────────────────┘
```

| 原则 | 说明 |
|------|------|
| **引擎优先** | 用户打开 App 的第一动作：右侧选 AI、发消息或跑命令 |
| 仓库在侧 | 左侧导航 + 文件结构；灵感从 Inbox/日记进入，不必先打开编辑器 |
| 编辑器是工作台 | 查看/修改引擎写回的内容，不是唯一起点 |
| 单 AI 单对话 | 每个 CLI 一条持续线程，不做多会话/对话树 |
| 输出必落盘 | 计划、复盘、graduate 等默认写进仓库对应目录，不只停在聊天里 |

## 引擎：13 条命令在做什么

命令是引擎的「标准化工序」，背后都是：**读仓库 → 加工 → 写回**。

| 类型 | 命令 | 引擎在帮你… |
|------|------|-------------|
| 日常 | /context /today /closeday /schedule | 从仓库拉上下文，产出今日计划与复盘 |
| 思考 | /trace /connect /challenge /ghost | 在仓库里溯源、找联系、压测观点、代你起草 |
| 灵感 | /ideas /graduate /drift /emerge | 扫仓库找模式、升级碎片为永久笔记、发现潜在项目 |
| 维护 | /sync | 把进展同步到 Master Control 等指挥文件 |

Prompt 细节见工作区 `AI_Commands_Prompt.md`；产品层只需记住：**引擎 = 对话 + 这些命令**。

## Agent CLI 自动发现（2026-06）

MetaMates **不要求用户手动配置 PATH**。应用启动时由主进程 `AcpDetector` 扫描本机已安装的 ACP CLI，结果显示在右侧 Agent 工具栏。

| 能力 | 说明 |
|------|------|
| 注册表 | `electron/shared/acpRegistry.ts` — 16+ CLI 及各自 ACP 启动参数 |
| 环境 | `shellEnv.getEnhancedEnv()` — 合并用户/系统 PATH、npm 全局、nvm/Scoop 等 |
| 检测 | PATH（`where` + `Get-Command`）→ npm 全局 bin → `npm list -g` |
| 启动 | 与检测共用 enhanced env；npx 桥接（Claude/Codex/CodeBuddy） |
| 刷新 | 窗口重新聚焦时自动重扫（新装 CLI 无需重启） |

技术细节见 [AGENT_DETECTION.md](./AGENT_DETECTION.md)。

## AI 技术路线

- **唯一主路径**：Electron → ACP → 本机 CLI 子进程。
- **Legacy 内置 API**：已移除。唯一主路径为 Electron → ACP → 本机 CLI。
- **Ollama**：本地隐私选项，与云端 CLI 并列可选。

## 移动端策略

| 阶段 | 能力 |
|------|------|
| **现已** | `/mobile` 只读浏览、搜索 |
| **当前迭代** | **快速剪藏** → `POST /api/capture` 写入 `Inbox/*.md` |
| **近期** | 设置项「允许局域网访问 Vault API」，手机同 WiFi 可剪藏 |
| **不做（首版）** | 手机端完整 Agent 对话、完整编辑器 |

剪藏单条笔记格式：标题 + 正文 + 可选来源 URL + 时间戳，落入 `01_…/Inbox/`。

## 商业化（原则，不实现计费）

采用 **本地核心免费 + 未来 Pro** 心智，避免架构锁死：

- **免费/开源核心**：本地 Vault、ACP 对话、MCP stdio、Ollama。
- **未来 Pro 可能**：高级工作流、官方模板包、团队同步、托管（若做再做账号体系）。
- **首版坚持**：数据在本地、无强制云、无订阅弹窗。

## 对外叙事（README / 介绍可用）

**短版：**  
> MetaMates 是你的私人灵感仓库和思考引擎。灵感丢进本地文件夹；打开右侧和 AI 对话，让它帮你整理、关联、升级想法——成品写回笔记，不消失在聊天里。

**长版：**  
> 我们都有碎片想法：日记里一句话、Inbox 里一条剪藏、项目里几段草稿。MetaMates 用本地 Markdown 仓库把它们收好（私人灵感仓库），再用 AI 对话和 /today、/graduate 等命令把碎片变成计划与洞察（引擎）。不是又一个 ChatGPT 窗口，而是**读过你全部笔记的搭档**。

**对比 Obsidian（如需一句）：**  
> Obsidian 擅长存和连；MetaMates 在此基础上多了引擎——**对话即启动**，命令即工序，输出必落盘。
