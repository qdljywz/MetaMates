MetaMates 是本地桌面应用：左侧灵感仓库装碎片与日记，右侧思考引擎是日常起点（对话 + 15 条 slash 命令），中间编辑器查看写回的成品——计划与洞察落盘到 Markdown，不会消失在聊天里。 设置页中部分能力尚在实验阶段，手册已分「现在可用」与「实验功能」两档。

5 分钟上手 → 配置 AI 助手

应用内随时可点标题栏 ? 打开帮助中心，并一键打开本手册。

📁 私人灵感仓库 本地 Markdown 文件夹，碎片、日记、剪藏先放这里，您完全拥有。
🧠 思考引擎 右侧 AI 对话 + slash 命令，读过仓库后把计划与洞察写回文件。
⚡ 15 条方法论命令 从 /today 到 /intel，一条命令完成计划、复盘、情报整理，结果自动落盘。
01

快速开始

  1. 安装并打开 MetaMates 首次启动时选择或新建工作区文件夹。
  2. 连接 AI 助手 点击标题栏齿轮 → 设置 → Agent，按「安装 AI 助手」完成至少一个助手的安装与登录。
  3. 从思考引擎开始 在右侧思考引擎发消息或点命令按钮；成品在中间编辑器打开、修改。

05

设置 · 现在可用

当前版本（v0.1.0）的主线是：私人灵感仓库 + 中间编辑器 + 思考引擎对话 + 15 条 slash 命令写回。 下面这些是建议现在配置、且与主线直接相关的设置；其余带「实验性」标签的见下一节。

打开设置：点击标题栏右侧齿轮图标 → 修改后务必点「保存」。请先确认左侧已加载工作区（能看到 01_日记与计划 等文件夹）。
档位含义现在要不要配
稳定 主路径，文档步骤可照着做 是
实验 有入口、方向明确，未完整验收 可跳过，见下一节

推荐配置顺序（仅 2 步）

1安装 AI 助手（设置 → Agent）— 必做，否则思考引擎无法对话
2重新同步 CLI Skills — 建议，启用 /today 等 15 条命令

自动保存 稳定

编辑笔记时自动写入磁盘，一般保持开启即可。

  • 设置 →「自动保存」→ 开关拨到开 → 点「保存」
  • 验证：编辑笔记后切换标签页，内容应仍在

安装 AI 助手 必做

MetaMates 外接本机 AI 助手（Gemini、Claude、CodeBuddy 等，底层为 ACP CLI）。至少安装并登录一个，思考引擎才能对话与执行 slash 命令。

  • 设置 → Agent → 点击「安装 AI 助手」
  • 选择助手（建议先试 Gemini 或 CodeBuddy）→ 未安装则点安装
  • 在 PowerShell 完成登录（如 gemini、codebuddy、claude）
  • 回到 MetaMates，在右侧思考引擎顶部选择对应助手 pill
  • 验证：显示「已连接」后发送「你好」能收到回复
提示：登录弹窗仅在您主动选择 AI 助手或发送消息时出现，不会在启动时挡住界面。

重新同步 CLI Skills 建议

将 /today、/context 等 15 条技能写入工作区各助手目录；不覆盖您已修改的文件。

  • 确认工作区已打开 → 设置 → Agent →「重新同步 CLI Skills」
  • 验证：思考引擎输入框输入 / 应出现 15 条命令列表
工作区隔离：每个工作区有独立的 AI 助手聊天记录和 skills；切换文件夹后会加载对应历史。

外观与语言

设置弹窗上半部分，按需调整即可：

主题深色 / 浅色 / 跟随系统
主题风格Nordic、Forest 等配色
字号编辑器字体大小
语言界面与初始化模板（中/英）

配置出问题时

AI 助手连不上

先在 PowerShell 单独运行对应助手 CLI 确认能登录；再在设置 → Agent查看安装状态，或点击思考引擎内的「安装 AI 助手」。

slash 命令不出现

在设置 → Agent 执行「重新同步 CLI Skills」，并确认当前工作区路径正确。

06

实验性功能 · 路线图

以下设置在应用里已有入口和部分实现，但尚未作为 v0.1.0 的主线承诺，也未经完整验收。 您可以先跳过；等有需求时再按文档尝试，遇到问题属于预期范围。

说明：实验性功能可能变动、文档滞后或与界面不完全一致。正式支持前，请勿依赖它们完成关键工作流。

规划中的能力

  • 日历 (iCal) — 导入 .ics，状态栏显示今日日程，配合 /schedule
  • Vault API — 本地 HTTP 接口，供 Cursor / Claude Desktop 读写笔记
  • MCP 服务器 — 为思考引擎注入自定义工具
  • Ollama — 本地大模型，数据不出本机

日历 (iCal) 实验性

导入日程后，状态栏显示今日事件数；与 /today、/schedule 的深度联动待完善。

  • 从 Google / Outlook 导出 .ics
  • 设置 →「日历 (iCal)」→「浏览」选文件 → 保存
  • 验证：状态栏出现日历图标与事件数（若无效，属已知限制）

Vault API（MCP 桥接） 实验性

在 localhost 暴露工作区 HTTP API，供外部 AI 工具搜索/读写 Markdown。需 MetaMates 保持运行。

  • 设置 → 开启 Vault API → 端口默认 17333 → 保存
  • 验证：设置页显示「运行中：http://127.0.0.1:17333」

Claude Desktop / Cursor 连接示例（路径改为本机实际位置）：

{
  "mcpServers": {
    "metamates-vault": {
      "command": "node",
      "args": ["/scripts/vault-mcp-bridge.mjs"],
      "env": { "VAULT_API_URL": "http://127.0.0.1:17333" }
    }
  }
}

「移动阅读」「局域网访问」同理——可用但未全面验证，默认保持局域网关闭。

MCP 服务器 实验性

给思考引擎注入额外 MCP 工具。开启 Vault API 时会自动注入 metamates-vault。

  • 设置 →「管理 MCP 服务器」→ 添加命令与参数
  • 保存后会尝试重载 AI 助手会话（成功率因 MCP 而异）

Ollama 本地模型 实验性

连接本机 Ollama。设置页可检测在线状态；与思考引擎内 AI 助手 pill 的完整打通仍在路线图中。

  • 安装 Ollama 并 ollama pull llama3.2
  • 设置 → 开启 Ollama → 填地址与模型名 → 保存

08

常见问题

我的数据存在哪里？

全部在您选择的灵感仓库文件夹中，以 Markdown 格式保存。备份该文件夹即可。

AI 助手无法连接？

请确认已在电脑上安装对应助手并完成登录，然后在 MetaMates 设置 → Agent 中检查状态，或在右侧思考引擎点击「安装 AI 助手」。

笔记之间如何建立关联？

使用 [[笔记名称]] 创建双向链接，或使用 #标签 分类。关系图谱可可视化这些连接。

设置里的 Vault API、MCP、Ollama、日历要不要配？

v0.1.0 不必配。它们属于实验性功能，可等有需求再试；不影响灵感仓库编辑与思考引擎对话主流程。
