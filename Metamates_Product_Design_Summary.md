## 一、产品定位与核心理念

### 1.1 产品定义

**MetaMates = Meta（元认知）+ Mates（伙伴）**

**一句话定义（当前对外口径）**：
> MetaMates 是**私人灵感仓库 + 思考引擎**：碎片先进本地 Markdown 仓库，右侧和 AI 助手对话或跑 `/today` 等命令——成品写回笔记，不消失在聊天里。

**历史表述（本文件早期草稿，保留供参考）**：
> Metamates 是你的**第二大脑架构师**，基于科学原则和 AI 原生设计，帮助你看见思维模式、连接知识碎片、规划未来行动，让每一次思考都更有价值。

### 1.2 核心理念

**"构建你的第二大脑，零配置、AI原生、本地优先，让每一次思考都更有价值"**

#### 五大核心价值：

1. **看见**
   - 看见你的思维模式
   - 看见知识的隐含关联
   - 看见你的成长轨迹

2. **连接**
   - 连接过去的经验
   - 连接相关的想法
   - 连接洞察与行动

3. **规划**
   - 基于科学原则规划未来
   - 确保规划有据可依
   - 连接过去和未来

4. **行动**
   - 将洞察转化为计划
   - 跟踪执行进度
   - 持续优化决策

5. **验证**
   - 确保操作可靠
   - 避免静默失败
   - 提供明确反馈

#### 核心设计原则：

**从经典时间管理原则融入**：
1. **优先级原则**：基于艾森豪威尔矩阵，优先处理重要任务
2. **精力原则**：根据精力状态安排任务，高精力做高价值工作
3. **专注原则**：保护深度工作时间，避免碎片化
4. **平衡原则**：平衡深度工作和浅层工作，平衡工作和生活
5. **持续改进原则**：定期审计时间使用，持续优化策略

**从竞品设计理念融入**：
1. **AI 原生**：所有功能基于 AI，自动化一切可以自动化的
2. **零配置**：开箱即用，智能默认，减少用户配置负担
3. **本地优先**：数据在本地，隐私保护，用户完全控制数据
4. **智能提醒**：主动服务，避免遗漏，智能跟进任务
5. **模块化**：灵活组合，适应不同场景，支持命令组合

### 1.3 产品个性

Metamates 不是：
- ❌ 复杂的知识管理系统
- ❌ 重量级的笔记软件
- ❌ 企业级的协作工具

Metamates 是：
- ✅ **第二大脑架构师**：主动构建你的知识体系
- ✅ **基于科学原则的时间管理顾问**：不仅仅是工具，而是基于经典时间管理原则的智能顾问
- ✅ **AI 原生助手**：所有功能基于 AI，自动化一切可以自动化的
- ✅ **零配置伙伴**：开箱即用，智能默认，减少用户配置负担
- ✅ **本地隐私守护者**：数据在本地，隐私保护，用户完全控制数据

### 1.4 目标用户

**核心用户画像**：

1. **个人创作者**
   - 需求：管理创作灵感，规划内容输出
   - 痛点：想法碎片化，难以系统化
   - 价值：发现创作模式，规划创作路径

2. **知识工作者**
   - 需求：整理工作笔记，提升决策质量
   - 痛点：信息过载，难以提炼洞察
   - 价值：识别关键信息，支持决策

3. **自我成长者**
   - 需求：记录成长轨迹，优化个人发展
   - 痛点：缺乏反思工具，难以持续改进
   - 价值：看见成长模式，规划下一步

### 3.2 核心概念

#### 概念 1：Master Control（主控中心）

**定义**：
- 存放在 `Daily Note&Plan/` 文件夹下
- 以日期命名：`YYYY-MM-DD PLAN.md`
- 当天的最高主控计划

**作用**：
- 作为当天的上下文基石
- 实时跟踪任务状态
- 确保计划与执行一致

**与命令的关系**：
- `/today` 读取 Master Control
- `/closeday` 更新 Master Control
- `/sync` 同步 Master Control

#### 概念 2：动态发现

**定义**：
- 自动搜索项目笔记、README、PRD
- 支持子文件夹特定指令
- 优先级：子文件夹 > 根项目

**作用**：
- 避免硬编码
- 支持多项目管理
- 提升上下文理解准确性

**与命令的关系**：
- `/context` 使用动态发现

#### 概念 3：验证与终结

**定义**：
- Act & Verify：操作后必须验证
- 禁止静默失败
- 提供验证声明

**作用**：
- 确保操作可靠性
- 及时反馈问题
- 提升用户信任

**与命令的关系**：
- `/sync` 包含验证机制

### 3.3 功能详细定义

#### 📂 日常管理类

**1. /context - 加载上下文（含动态发现）**
- **功能**：
  - 读取最近的项目、反思和优先级
  - 自动搜索项目笔记、README、PRD
  - 支持子文件夹特定指令
- **场景**：开始工作时，快速了解当前状态
- **价值**：减少认知负担，快速进入心流状态
- **Prompt**：
  ```
  Read my vault and summarize my current context. Include:
  1. Active projects (from README.md, PRD, or project notes)
  2. Recent reflections (from daily notes)
  3. Priorities (from Master Control file)
  4. Project-specific instructions (from GEMINI.md in subdirectories)
  5. Time-sensitive tasks and deadlines
  ```

**2. /today - 今日计划（基于Master Control）**
- **功能**：
  - 优先读取今天的 Master Control 文件
  - 根据日记、日历和任务列表生成计划
  - 考虑能量水平和时间块
- **场景**：每天早上规划当天任务
- **价值**：明确每日重点，提高执行力
- **Prompt**：
  ```
  Read today's Master Control file (YYYY-MM-DD PLAN.md) if it exists.
  If not, generate a prioritized plan for today based on:
  1. Yesterday's carryover tasks
  2. This week's goals
  3. Calendar events
  4. Energy levels and time blocks
  Update the Master Control file with the plan.
  ```

**3. /closeday - 每日复盘（含昨夜预谋）**
- **功能**：
  - 总结当天进展
  - 记录新想法
  - 列出需要结转到明天的任务
  - 生成明天的 Master Control 文件
- **场景**：每天结束时复盘当天工作，并规划明天
- **价值**：沉淀经验，持续改进，提前规划
- **Prompt**：
  ```
  Review today's progress and:
  1. Summarize key achievements
  2. Capture new ideas and insights
  3. Note unfinished tasks
  4. Update task status in Master Control file
  5. Generate tomorrow's Master Control file (昨夜预谋)
  6. Provide time calibration check
  ```

**4. /schedule - 时间调度**
- **功能**：根据优先级和日历，建议本周的时间分配方案，标记潜在冲突
- **场景**：规划一周时间安排
- **价值**：优化时间分配，避免冲突
- **Prompt**：`Based on my current projects and priorities, suggest a schedule for this week. Flag any conflicts between what I say matters and how I'm spending time.`

---

#### 🧠 深度思考类

**5. /trace - 溯源想法**
- **功能**：追踪某个特定想法在整个库中是如何演变的轨迹
- **场景**：理解一个想法的起源和发展
- **价值**：理解思维路径，避免重复思考
- **Prompt**：`Track how a specific idea has evolved over time across my Obsidian vault. Take a topic as input, search for all mentions, follow backlinks, and output a timeline.`

**6. /connect - 寻找连接**
- **功能**：寻找两个看似无关的主题之间的意外联系
- **场景**：发现不同主题之间的关联
- **价值**：跨领域创新，拓展思维广度
- **Prompt**：`Find connections between [topic A] and [topic B] in my vault. Show me the notes that link them and any patterns you see.`

**7. /challenge - 挑战观点**
- **功能**：压力测试观点，寻找矛盾点或潜在错误假设
- **场景**：验证一个观点的正确性
- **价值**：避免确认偏误，拓展思维边界
- **Prompt**：`Review my notes on [topic]. Where am I contradicting myself? What assumptions am I making that might be wrong?`

**8. /ghost - 模拟代写**
- **功能**：模拟用户的口吻和价值观来起草文档或回复
- **场景**：需要以自己的风格撰写内容
- **价值**：保持个人风格，提高写作效率
- **Prompt**：`Based on my vault, how would I answer this question: [question]? Use my voice and reference specific notes where relevant.`

---

#### 💡 灵感挖掘类

**9. /ideas - 点子报告**
- **功能**：扫描库中新兴模式，生成包含工具、人脉和写作主题的报告
- **场景**：寻找新的机会和方向
- **价值**：发现隐性机会，拓展可能性
- **Prompt**：`Scan my vault for emerging patterns. Generate ideas for: tools I should build, people I should reach out to, topics I should investigate, and things I should write.`

**10. /graduate - 灵感升级**
- **功能**：从碎片化日记中提取灵感，转化为独立的永久笔记
- **场景**：将零散的想法系统化
- **价值**：将碎片转化为知识，提升价值
- **Prompt**：`Scan my daily notes from the past 14 days. Find ideas that deserve their own note and create standalone files for them.`

**11. /drift - 潜意识漂移**
- **功能**：捕捉在不同笔记中反复出现但尚未成形的关键词或主题
- **场景**：发现潜意识关注的焦点
- **价值**：看见隐性关注点，发现新方向
- **Prompt**：`Scan my vault for recurring themes or phrases that appear across unrelated notes. What ideas am I drifting toward without realizing it?`

**12. /emerge - 项目涌现**
- **功能**：识别正在聚合而成的潜在项目、文章或产品雏形
- **场景**：发现可以孵化的新项目
- **价值**：发现潜在机会，加速项目孵化
- **Prompt**：`Find clusters of related ideas in my vault that could become a project, essay, or product.`

---

#### � 规划基础类

**13. /sync - 规划基础（含验证）**
- **功能**：
  - 总结最新资讯和进展
  - 同步关键信息到 Master Control
  - 为后续规划提供依据
  - 验证操作结果
- **场景**：
  - 每日结束时，总结今日进展，生成明日规划基础
  - 项目关键节点，总结项目进展，生成下一步规划基础
  - 周期性回顾，总结阶段成果，生成下一阶段规划基础
- **价值**：
  - **总结过去**：扫描全库进展，复盘今日关键信息
  - **同步现在**：更新 Master Control，确保信息最新
  - **规划未来**：生成明天的 Master Control，为后续规划提供依据
  - **验证结果**：确保操作可靠，提供验证报告
- **核心作用**：连接过去和未来，是时间规划和事项规划的基础
- **Prompt**：
  ```
  Synchronize the entire vault:
  1. Update Master Control file with today's progress
  2. Verify recent file operations
  3. Check for conflicts or inconsistencies
  4. Update project status in README files
  5. Generate a sync report with verification evidence
  ```

## 五、模板系统设计

### 5.1 模板系统理念

**核心价值**：
- **自动化**：自动填充格式化内容，减少用户手动操作
- **美观性**：格式化输出，提升阅读体验
- **一致性**：统一格式，便于回顾和比较
- **效率**：快速开始，降低使用门槛

**设计原则**：
1. **零配置**：模板自动应用，无需用户手动选择
2. **智能填充**：基于上下文自动填充内容
3. **格式美观**：Markdown 格式，支持富文本显示
4. **灵活定制**：支持用户自定义模板
